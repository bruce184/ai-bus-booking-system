# UI Design Context

## Product Feel

The app is an operational travel booking system. It should feel clear, fast, and trustworthy.

## Required Screens

- Search page
- Search results
- Trip detail and seat map
- Checkout
- Payment simulation
- Booking confirmation
- Booking lookup
- Customer booking history
- Admin dashboard
- Admin routes/stops/vehicles/trips
- Admin bookings/check-in
- Chatbot panel

## UI Rules

1. Show loading, empty, error, success, and expired hold states.
2. Seat map must make four states obvious: available, held, booked, blocked.
3. Checkout must show hold countdown.
4. Use compact admin tables and filters for repeated operational work.
5. Do not hide required booking lookup privacy fields.
6. Keep mobile usable for customer flow.
7. Keep admin dashboard optimized for laptop/desktop.
